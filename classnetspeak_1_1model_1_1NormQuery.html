Detailed Description

A norm query (= normalized query) is a Netspeak query that only contains one concatenation. This concatenation is only allowed to contain question mark wildcards and constant words.

This represents the type of query a Netspeak index can answer without any further query-processing and can be thought of as the "native language" of a Netspeak Index.

Since a Netspeak query typically generates many norm queries, each unit of a norm query has an id of the token in the original Netspeak query that generated it. This can be used to track which word is matched by which token of the original Netspeak query.
